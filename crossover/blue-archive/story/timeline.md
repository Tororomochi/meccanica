# Blue Archive クロスオーバー タイムライン

> Meccanica crossover reference.
>
> ブルーアーカイブ側クロスオーバーのみの時系列整理。ドルフロ本編側・グリフィン本編側の詳細時系列は扱わない。
>
> 原初情報、年齢、旧素性、帰還事故、2025年関連情報には触れない。

## 運用方針

- このファイルは、Blue Archive クロスオーバー内で起きた出来事の順番を整理する。
- 詳細なあらすじは `story/` 配下の個別ファイルへ分ける。
- 学校・部活・キャラクターの設定は各学校ディレクトリへ分ける。
- 日付ではなく、物語上の順番で管理する。
- 追加時は `Txx` の番号を増やす。
- 章ファイル名は `01-english-slug.md`、`02-english-slug.md` のように、2桁番号 + 英語スラッグで統一する。
- 各 `Txx` の「関連資料」には、対応する章ファイルへのリンクを必ず入れる。
- 章ファイルを追加した場合は、`timestamp.md`、この `timeline.md`、必要に応じて `README.md` を更新する。

### T06: Scadente Muro / D-SIS の提示

- アンナがセミナーへ書類処理支援・監査隔離システム「Scadente Muro」を持ち込む。
- 売り方が「600万円の魔法のUSB」なので、ユウカに強く警戒される。
- 実態はAIに決裁させるものではなく、分類、照合、不備検出、期限警告、ログ化を補助する隔離・監査型の業務基盤。
- 正式名称候補として D-SIS（Document Shield Isolation System）が提案される。
- 試験対象は食堂改善関連書類のみで、個人情報、成績、懲戒、本予算決裁、軍機、MODY情報、グリフィン内部情報は対象外とされる。

関連資料:

- `01-millennium-d-sis-arc.md`

### T07: ヴェリタス隔離検証

- ユウカがヴェリタスへD-SISの隔離検証を依頼する。
- アンナ、ユウカ、ヒマリがヴェリタス部室を訪れる。
- チヒロ、ハレ、マキが検証に参加する。
- 通常端末への接続禁止、隔離環境での開封、外部通信なし、ログ全保存、ロールバック可能という条件で検証が始まる。
- 添付偽装、過剰権限要求、ログ削除誘導、業務文書内命令混入、悪性添付混入などが試験される。

関連資料:

- `01-millennium-d-sis-arc.md`

### T08: 運用ミス試験と Meta-Audit Mode

- アンナは、システムへの攻撃だけでなく、人間の運用ミスこそ危険だと指摘する。
- 添付ファイル間違い、承認先の誤指定、同名書類の重複提出、アレルゲン表示漏れ、警告無視などの運用ミス試験が提案される。
- ヴェリタスが隔離仮想環境で遊び始めないよう、探索的テストは目的、範囲、期待挙動、停止条件、ログ保存先、担当者を記録した上で許可制となる。
- 検証者自身の作業ログを監査する Meta-Audit Mode が追加される。

関連資料:

- `01-millennium-d-sis-arc.md`

### T15: レッドウィンター酪農支援とヒマリ外骨格編

- 休暇中のメカニカは、シャーレから機密文書輸送を依頼され、先生と共にレッドウィンターへ向かう。
- レッドウィンターでは酪農部設立支援契約が結ばれ、チェリノが自分で署名・押印し、赤い温度調整ウツボジャケットを受け取る。
- キヴォトスゲキサムチキン、キヴォトスゲキサムカミカゼウシ、キヴォトスゲキサムオオカミモドキなど、寒冷地家畜と外敵の生態が確認される。
- メカニカは飼育現場の防具運用を観察し、新人は防具、ベテランは防具なし、定期的に防具への馴化を行う改善案を出す。
- 契約後、機密文書を狙う爆発物事件が発生し、メカニカは本物の文書をポータルへ退避、空のアタッシュケースを囮にして車列を動かす。
- トリニティ、アリウス、SRT RABBIT小隊、便利屋68がそれぞれ別経路で爆発物確認・安全化・ルート確保に関与する。

- シャーレ帰還後、メカニカは文書を先生の執務室へ戻し、輸送契約満了を宣言する。
- ミレニアムでは、エンジニア部がメカニカの外回り用足部モジュールを非破壊検査し、そのデータを元にヒマリ用歩行補助外骨格が製作される。
- 三日後、ヒマリ用外骨格は補助付き直立、30m歩行、旋回、物体キャッチ、初期学習ノイズ除去まで成功する。
